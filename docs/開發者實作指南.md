# ⚙️ Project Golem 開發者實作指南

> 最後更新：2026-03-04

## 一、新增技能 (Step-by-Step)

### 1.1 建立技能檔案

在 `src/skills/user/` 目錄下建立新 `.js` 檔案：

```javascript
// src/skills/user/my-skill.js

module.exports = {
    name: "my_skill",              // AI 呼叫的名稱 (底線格式)
    description: "這個技能做了 X 事情",
    version: "1.0",

    async run(ctx) {
        const args = ctx.args || {};   // AI 傳入的參數
        const brain = ctx.brain;       // GolemBrain 實例（可調用 brain.sendMessage）

        // 你的邏輯...
        const result = await doSomething(args.param);

        return `✅ 完成：${result}`;   // 回傳給 AI 的執行結果
    }
};
```

### 1.2 在技能書中描述給 AI

在 `src/skills/lib/` 建立對應 `.md` 說明文件，Golem 啟動時會讀取並注入 Gemini：

```markdown
## my_skill - 我的技能

**觸發時機**：當用戶要求 X 操作時

**呼叫格式**：
[GOLEM_ACTION]
{"action": "my_skill", "param": "值"}
[/GOLEM_ACTION]

**參數說明**：
- `param`：目標值
```

### 1.3 熱重載（不需重啟）

透過 Telegram 指令：
```
/skills     ← 確認技能已載入
```

或透過程式碼：
```javascript
skillManager.refresh();
```

## 二、Golem Protocol 回應格式

AI 回應必須包含以下結構標記，`NeuroShunter` 才能正確解析：

### 2.1 回覆用戶

```
[GOLEM_REPLY]
這是顯示給用戶的回覆文字
[/GOLEM_REPLY]
```

### 2.2 寫入長期記憶

```
[GOLEM_MEMORY]
用戶偏好使用 Dark Mode。
用戶的伺服器 IP 是 192.168.1.100。
[/GOLEM_MEMORY]
```

### 2.3 執行技能或動作

```
[GOLEM_ACTION]
[
  {"action": "skill_name", "param1": "value1"},
  {"action": "another_skill"}
]
[/GOLEM_ACTION]
```

### 2.4 自主介入（突破靜默模式）

在回覆中加入 `[INTERVENE]` 標籤，即可在觀察者模式下強制回覆：

```
[GOLEM_REPLY]
[INTERVENE] 注意！我偵測到此訊息有事實錯誤，需要糾正...
[/GOLEM_REPLY]
```

### 2.5 Multi-Agent 任務委派

```
[GOLEM_ACTION]
[{
  "action": "multi_agent",
  "target": "golem_B",
  "message": "請幫我分析這份報告"
}]
[/GOLEM_ACTION]
```

## 三、新增 golems.json 設定

```json
[
  {
    "id": "golem_A",
    "tgToken": "你的_TG_BOT_TOKEN",
    "role": "主要對外窗口",
    "tgAuthMode": "ADMIN",
    "adminId": "你的_TG_USER_ID"
  },
  {
    "id": "golem_B",
    "tgToken": "另一個_TG_BOT_TOKEN",
    "role": "群組客服機器人",
    "tgAuthMode": "CHAT",
    "chatId": "-100群組ID"
  }
]
```

| `tgAuthMode` | 行為 |
|-------------|------|
| `ADMIN` | 只回應 `adminId` 的訊息 |
| `CHAT` | 回應指定 `chatId` 群組的所有成員訊息 |

## 四、記憶系統整合要點

### 4.1 在技能中讀寫記憶

```javascript
// 寫入長期記憶
await brain.memorize("用戶喜歡簡潔的回答", { type: 'preference' });

// 召回相關記憶
const memories = await brain.recall("用戶喜好");
// → [{ text: "用戶喜歡簡潔的回答", metadata: {...} }]
```

### 4.2 在技能中寫入對話日誌

```javascript
// 通常不需要手動寫入，ConversationManager 會自動記錄
// 若需要從技能內手動記錄特殊事件：
brain._appendChatLog({
    sender: 'System',
    content: '技能執行了某重要操作',
    type: 'system'
});
```

### 4.3 LOG_BASE_DIR 使用原則

> [!IMPORTANT]
> 技能或任何直接使用 `ChatLogManager` 的程式碼，**絕對不可**硬寫日誌路徑。應從 config 取得：
>
> ```javascript
> const { LOG_BASE_DIR, GOLEM_MODE } = require('../../config');
> // 或者優先使用 brain.chatLogManager（路徑已正確初始化）
> ```

## 五、ResponseParser 解析邏輯

`src/utils/ResponseParser.js` 用於解析 AI 的結構化輸出：

```javascript
const parsed = ResponseParser.parse(rawAiResponse);
// → {
//     reply: "顯示給用戶的文字",
//     memory: "要記憶的內容 (或 null)",
//     actions: [{ action: "...", ...params }]
//   }
```

解析順序（Priority）：
1. 提取 `[GOLEM_REPLY]...[/GOLEM_REPLY]` → `parsed.reply`
2. 提取 `[GOLEM_MEMORY]...[/GOLEM_MEMORY]` → `parsed.memory`
3. 提取 `[GOLEM_ACTION]...[/GOLEM_ACTION]` → JSON 解析成陣列 → `parsed.actions`
4. 若無任何標記，將整個回應視為 `reply`（向下相容）

## 六、DOMDoctor 自動修復機制

GolemBrain 依賴 Puppeteer 操作 Gemini 網頁，當 Google 更新 UI 導致 Selector 失效時，`DOMDoctor` 會：

1. 偵測到操作失敗 → 拋出 `SELECTOR_HEALED:<type>:<new_selector>` 錯誤
2. `GolemBrain.sendMessage()` 捕捉此錯誤 → 自動更新 Selector 緩存
3. 將新 Selector 持久化到磁碟（`selectors.json`）
4. 用新 Selector 重試操作

> 這使得系統能在 Google 小幅更新 UI 時**自動癒合**，不需人工介入。

## 七、常見問題 (FAQ)

**Q: 新增的技能沒有被 AI 使用到**

AI 只會在技能書（`src/skills/lib/*.md`）有描述時才知道如何呼叫。確認有對應的 `.md` 說明文件，並在 Telegram 發送 `/new` 重新注入記憶。

**Q: 技能執行後沒有回應**

技能的 `run()` 函數必須回傳字串。確認 `return` 有值，且未拋出未捕捉的例外。

**Q: SINGLE 和 MULTI 模式的日誌分開了嗎？**

是的。`GOLEM_MODE=SINGLE` → `logs/single/`；未設定或有 `golems.json` → `logs/multi/<golemId>/`。兩套資料完全隔離，切換模式後記憶不互通。

**Q: 如何在沒有 Telegram 的環境下測試？**

可直接實例化 `GolemBrain` 並呼叫 `init()` 後使用 `sendMessage()` 進行測試，不需要 Bot Token。

**Q: SkillArchitect (`/learn`) 生成的技能有安全疑慮嗎？**

生成的技能在匯出時不能包含 `child_process`、`exec(`、`spawn(` 等敏感調用，`SkillManager.importSkill` 有靜態掃描過濾。核心技能 (`src/skills/core/`) 無法被匯出分享。
